# 福源登山酒店综合管理系统 — 开发需求文档 v3.0

> 项目代号：Hotel Manager  
> 技术栈：Vue 3 + Vite + Express + SQLite (sql.js)  
> 数据库：单文件 SQLite（`data/hotel.db`，支持自动备份）  
> 目标平台：移动端 H5（响应式适配手机/平板/电脑）  
> 业务范围：客房（42间）+ 餐厅（对外营业）+ 请香 + 财务

---

## 一、项目概述

### 1.1 定位
福源登山酒店的一体化运营管理工具，覆盖前台、餐厅、财务三大业务线：

- **前台**：房态管理 / 入住退房 / 预订 / 清洁维修 / 客人档案
- **餐厅**：菜品管理 / 点单结账 / 挂房账 / 外客收银
- **请香**：商品管理 / 销售录入 / 返佣计算 / 库存管理
- **财务**：支出管理（含单据拍照）/ 日报自动汇总 / 押金管理 / 退房合并结算

### 1.2 核心原则
1. **业务闭环**：餐饮/请香挂房账 → 退房合并结算 → 押金抵扣 → 开票，全链路跑通
2. **自动汇总**：日报表从各业务模块自动拉取数据，无需手动录入
3. **状态驱动**：房间状态决定可操作（如入住房间不可删除）
4. **数据安全**：每小时自动备份数据库，保留30天

## 四、业务流程

### 4.1 标准入住流程
```
空房 → 办理入住（录入客人/日期/押金/渠道）
     → 订单：已入住 + 房间：已入住 + 客人：自动保存/升级
     → 押金：自动创建 deposit_records（收押金）
```

### 4.2 预订→入住流程
```
空房 → 仅预订 → 订单：已预订 + 房间：已预订
     → 客人到店 → 订单页「🛎️ 办理入住」
     → 订单：已入住 + 房间：已入住
```

### 4.3 退房→清洁→空房（含合并结算）
```
已入住 → 退房结账
       → 自动汇总：房费 + 挂房账餐饮 + 挂房账请香
       → 押金处理：全额退/抵扣/部分退
       → 确认结账
       → 订单：已完成 + 房间：清洁中
       → 挂账订单标记：已结算
       → 创建清洁记录
```

### 4.4 餐饮点单→结账
```
住店客人：
  点单 → 结账（选挂房账）→ 关联到 booking
  → 退房时合并结算

外客：
  点单 → 结账（选现金/微信/支付宝）→ 即时入账
  → 日报自动计入餐饮收入
```

### 4.5 请香销售→挂账
```
登记销售 → 选商品+数量
         → 住店客：挂房账 → 退房时结算
         → 外客：直接结账
         → 自动扣减库存 + 计入日报
```

### 4.6 日报自动生成
```
每天打开日报页 → 系统自动汇总：
  ① bookings(当日退房)  → 客房收入
  ② catering_orders(当日结账) → 餐饮收入
  ③ incense_sales(当日) → 请香收入
  ④ expenses(当日)      → 总支出
  ────────────────────────
  ⑤ 写入 daily_reports   → 展示经营汇总
```

## 九、数据库备份方案

```batch
@REM backup-hotel.bat — 每小时执行一次（Windows计划任务）
@echo off
set SRC=C:\Users\Administrator\workspace\hotel-manager\data\hotel.db
set DST=C:\Users\Administrator\workspace\hotel-manager\backup\
set DATETIME=%date:~0,4%%date:~5,2%%date:~8,2%_%time:~0,2%
set DATETIME=%DATETIME: =0%
mkdir %DST% 2>nul
copy %SRC% %DST%\hotel_%DATETIME%.db

@REM 保留最近30天，删除更早的备份
forfiles /p %DST% /m *.db /d -30 /c "cmd /c del @path" 2>nul
```
