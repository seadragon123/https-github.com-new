## 1. 系统概述

### 1.1 系统简介

**酒店管家**是一套专为民宿/中小型酒店设计的一体化管理系统，覆盖客房管理、预订订单、餐饮点单、请香销售、财务日报、动态房价等全业务流程。系统为 42 间客房规模定制，支持多用户分角色协作。

### 1.2 访问方式

- **系统地址**：`https://hotel-manager-api-production.up.railway.app`
- **备选地址**：`https://wuzhou.netlify.app`（如已部署）
- **浏览器要求**：手机端微信/Chrome/Safari、电脑端 Chrome/Edge 均可

### 1.3 技术栈

前端 Vue 3 + Vant UI 移动优先设计，后端 Express + sql.js，适配手机和电脑（≥768px 显示 PC 布局）。

## 4. 房间管理

### 4.1 功能概述

管理全部 42 间客房的房态信息，支持按状态筛选、增删改房间、查看房间详情。

### 4.2 房态筛选

顶部标签栏切换房间状态：

| 状态 | 说明 |
|------|------|
| 全部房间 | 显示所有房间 |
| 已入住 | 当前有客人入住 |
| 清洁中 | 退房后正在清洁 |
| 维修中 | 有故障待维修 |
| 已预订 | 已被预订未入住 |
| 空房 | 空闲可售 |

### 4.3 房间卡片

每个房间显示：

- 房号 + 楼层 + 房型
- 当前状态（彩色标签）
- 当前入住客人姓名（如有）
- 房价（¥/晚）
- 维修中的房间显示 ⚠️ 维修说明

点击卡片 → 进入房间详情页。

### 4.4 新增房间

点击右上角「+ 新增房间」按钮，填写：

| 字段 | 必填 | 说明 |
|------|:----:|------|
| 房号 | ✅ | 如 401，需唯一 |
| 楼层 | ❌ | 默认 1 |
| 房型 | ✅ | 选：标准大床房/标准双床房/豪华大床房/豪华双床房/豪华套房 |
| 价格 | ✅ | ¥/晚 |
| 备注 | ❌ | 如"朝南，采光好" |

### 4.5 房间详情

从房间列表点击具体房间进入，可查看该房间的：

- **基本信息**：房号、房型、楼层、价格、描述
- **当前状态**：可手动变更（一键切换状态）
- **当前订单**：该房间的实时入住/预订信息
- **操作按钮**：维护记录、清洁记录

## 5. 订单管理

### 5.1 功能概述

管理所有客人的预订和入住订单，支持按状态/渠道筛选、编辑订单、直接入住、取消订单。

### 5.2 订单筛选

| 筛选项 | 说明 |
|--------|------|
| 全部 | 所有状态的订单 |
| 已入住 | 正在入住中的订单 |
| 已预订 | 已预订但未入住的订单 |
| 已完成 | 已退房结账的订单 |
| 渠道 | 散客/携程/美团/飞猪/协议/陪护 |

### 5.3 订单卡片

每个订单显示：

- 房号 + 状态标签 + 渠道标签（如"携程"）
- 订单金额
- 客人姓名 + 电话
- 入住日期 → 退房日期 + 房型
- 操作按钮（根据状态不同）

### 5.4 订单操作

**已入住状态**：
| 操作 | 说明 |
|------|------|
| 退房结账 | 跳转到退房结算页面 |
| 取消订单 | 取消并释放房间 |

**已预订状态**：
| 操作 | 说明 |
|------|------|
| 直接入住 | 无需到前台即可标记入住 |
| 取消 | 取消预订并释放房间 |

### 5.5 编辑订单

点击订单上的 ✏️ 按钮，可编辑：

- 客人姓名 / 联系电话
- 入住/退房日期
- 渠道
- 房费金额
- 备注

### 5.6 新增订单

在房间详情页或首页快速操作中可新增订单。

## 6. 退房结账

### 6.1 功能概述

这是系统最核心的模块之一，支持**合并结算**——将房费、餐饮挂账、请香挂账在同一页面完成结账，同时处理押金。

### 6.2 进入方式

- 订单管理 → 点击「退房结账」按钮
- 首页 → 快速操作「退房结账」→ 选择待退房订单

### 6.3 结算页面内容

**订单信息区**：
- 房间号 + 房型
- 客人姓名
- 入住/退房日期
- 渠道来源

**费用结算区**：

| 费用项 | 说明 |
|--------|------|
| 🏨 房费（N晚） | 系统根据房价×住夜数自动计算 |
| 🍜 餐饮挂账 | 该客人在餐厅消费且挂房账的订单，自动汇总 |
| 🪷 请香挂账 | 该客人请香且挂房账的销售，自动汇总 |
| 应付合计 | 三项加总 |

**押金处理**：
显示当前押金余额，三种处理方式：

| 方式 | 说明 |
|------|------|
| 从押金抵扣消费 | 押金自动抵扣部分或全部应付金额 |
| 全额退还押金 | 押金全部退还，房费另收 |
| 部分退还 | 自定义退还金额 |

**结账完成后**：
- 订单自动标记为「已完成」
- 房态自动变为「清洁中」
- 餐饮挂账的订单自动标记为已结账
- 请香挂账记录自动标记

## 7. 餐厅管理

### 7.1 功能概述

管理餐厅的菜品、堂食/外卖点单、结账。支持"挂房账"——客人在餐厅消费后记到客房账单，退房时合并结算。

### 7.2 今日营业概况

| 指标 | 说明 |
|------|------|
| 堂食 | 今日堂食单数 |
| 外卖 | 今日外卖单数 |
| 挂账 | 今日挂房账单数 |
| 总单数 | 全部订单数 |
| 今日营收 | 今日已结账金额合计 |

### 7.3 菜品管理

**新增菜品**：点击「+ 新增」→ 填写名称、分类（热菜/凉菜/主食/酒水/汤品）、价格、状态（上架/下架）

**分类筛选**：全部 / 热菜 / 凉菜 / 主食 / 酒水 / 汤品

**编辑/删除**：点击菜品卡片上的 ✎ / ✕

**下架**：点击菜品卡片切换上架/下架状态，下架后不出现在点单菜单中

### 7.4 快速点单

点击「+ 快速点单」→ 弹出点单面板：

1. 选择订单类型：堂食 / 外卖
2. 输入客人姓名 + 电话（选填）
3. 选择菜品（点击 + / - 调整数量）
4. 如客人住店且想挂账→打开「挂房账」开关→选择对应房间/订单
5. 提交订单

### 7.5 结账

订单列表中的「就餐中」订单可点击结账：

1. 选择支付方式：现金 / 微信 / 支付宝 / 挂房账
2. 确认金额
3. 完成结账

### 7.6 销量排行

切换到"销量排行"标签，可查看：

- **统计周期**：全部 / 本月 / 本周
- **排行信息**：排名、菜品名称、分类、销量、销售额
- 前三名有 🥇🥈🥉 标记

## 8. 请香管理

### 8.1 功能概述

管理请香商品的销售和库存。支持挂房账——客人请香后金额记入客房账单，退房时统一结算。支持销售返佣计算。

### 8.2 今日销售概览

| 指标 | 说明 |
|------|------|
| 今日收入 | 今日请香销售总金额 |
| 返佣合计 | 今日需支付的返佣总金额 |
| 销售笔数 | 今日交易笔数 |

### 8.3 商品管理

**新增商品**：点击「+ 新增商品」→ 填写名称、价格、库存数量、单位（份/套/个）、上架状态

**编辑**：点击商品行 → 修改信息

**下架**：库存 ≤ 5 时自动红色警示，可手动下架

**删除**：点击 ✕

### 8.4 新增销售

点击「+ 新增销售」→ 填写：

| 字段 | 说明 |
|------|------|
| 商品 | 从下拉列表选择 |
| 数量 | 购买数量（自动减库存） |
| 金额 | 自动根据单价×数量计算 |
| 客人姓名 | 输入姓名 |
| 支付方式 | 现金/微信/支付宝/挂房账 |
| 是否返佣 | 开启后填写返佣比例（%） |
| 挂房账 | 如选挂房账，选择对应房间/订单 |

### 8.5 今日销售记录

列表显示当日所有销售记录，每条包含：

- 商品名称 + 数量
- 金额
- 支付方式
- 返佣信息（如有）
- 客人姓名

## 9. 营业报表

### 9.1 功能概述

每日营业数据的自动汇总和展示页面，分为四大板块：客房收入、餐饮收入、请香收入、支出。支持打印和导出 CSV。

### 9.2 使用流程

1. 选择**日期**（默认今天）
2. 输入**报告人**（签字）
3. 依次点击各板块的「🤖 刷新」按钮自动汇总数据
4. 核对后点击「🖨️ 打印」或「📥 CSV」导出

### 9.3 客房收入

点击「🤖 刷新」后，系统自动从当日已完成订单生成明细：

| 列 | 说明 |
|----|------|
| # | 序号 |
| 房号 | 房间号 |
| 渠道 | 散客/携程等 |
| 客户姓名 | 客人名称 |
| 金额 | 该笔房费 |
| 支付方式 | 现金/微信/支付宝 |

底部显示**客房收入合计**。

### 9.4 餐饮收入

点击「🤖 刷新」后，自动汇总当日已结账的餐饮订单：

| 列 | 说明 |
|----|------|
| 序号 | 序号 |
| 订单号 | 餐饮订单编号 |
| 客人 | 客人姓名 |
| 类型 | 堂食/外卖 |
| 金额 | 订单金额 |
| 支付方式 | 现金/微信/支付宝/挂房账 |

### 9.5 请香收入

点击「🤖 刷新」后，自动汇总当日请香销售记录：

| 列 | 说明 |
|----|------|
| 序号 | 序号 |
| 商品 | 请香商品名称 |
| 数量 | 销售数量 |
| 金额 | 销售金额 |
| 返佣 | 返佣金额 |
| 净收入 | 金额-返佣 |
| 支付方式 | 现金/微信/支付宝/挂房账 |

### 9.6 支出明细

点击「🤖 刷新」后，显示当日录入的支出记录：

| 列 | 说明 |
|----|------|
| 序号 | 序号 |
| 类别 | 日常耗材/餐饮成本/维修杂费等 |
| 金额 | 支出金额 |
| 说明 | 支出事由 |

### 9.7 日报汇总

页面底部自动汇总当日所有数据：

| 项目 | 说明 |
|------|------|
| 客房收入 | 自动汇总 |
| 餐饮收入 | 自动汇总 |
| 请香收入 | 自动汇总 |
| 支出合计 | 自动汇总 |
| **净现金流** | 收入-支出，自动计算 |

## 11. 房价日历

### 11.1 功能概述

可视化日历界面管理每日房价，支持按房型查看、单日调价和批量调价，支持周末/节假日标记。

### 11.2 操作界面

```
        ◀ 2026年6月 ▶
┌─────────────────────────────┐
│ 日  一  二  三  四  五  六  │
│                            │
│    1   2   3   4   5   6   │
│ ¥268  268  298  298  368   │
│                            │
│  7   8   9  10  11  12  13  │
│ ¥368  398  268  268  298   │
│  ...                       │
└─────────────────────────────┘
```

- **周末**：周六/日以紫色背景显示
- **节假日**：以特殊标记显示
- **今天**：绿色边框高亮
- **有价格**：显示当日价格

### 11.3 月份/房型切换

- ◀ 上月 / 下月 ▶：切换月份
- 房型下拉框：切换不同房型查看（标准大床房/标准双床房/豪华大床房/豪华双床房/豪华套房/全部）

### 11.4 单日调价

点击任意日期 → 弹出编辑窗口：

| 字段 | 说明 |
|------|------|
| 房价 | 修改该日该房型价格 |
| 标记 | 普通/节假日 |
| 标签文字 | 如"端午"、"五一"等 |

### 11.5 批量调价

点击「⚡ 批量调价」→ 弹出批量操作面板：

**模式**：

## 18. 常见问题

### Q1：登录提示「网络错误」

**原因**：前端页面成功加载，但登录请求发送到后端的 API 地址不对（如请求发到 Netlify 但没有后端服务）。

**解决**：
1. 确认访问的是 Railway 地址（`https://hotel-manager-api-production.up.railway.app`）
2. 如果是通过 `wuzhou.netlify.app` 访问，需要在 Netlify 设置环境变量 `VITE_API_BASE=https://hotel-manager-api-production.up.railway.app/api`

### Q2：数据会不会丢失？

系统使用 Railway Volume 持久化存储 SQLite 数据库。正常部署不会丢数据。如需备份，可通过 Railway 管理后台下载数据库文件。

### Q3：如何备份数据？

数据库文件位于 Railway Volume 的 `/app/data/hotel.db`。当前暂无自动备份，建议定期通过 Railway 后台 Volume 文件管理下载备份。

### Q4：结账时餐饮/请香挂账没有自动显示？

确认该客人的餐饮订单或请香销售已选择「挂房账」支付方式，且挂账到的房号/订单正确。

### Q5：押金怎么处理？

押金在「退房结账」页面统一处理：
- 可抵扣应付消费
- 可全额退还
- 可部分退还

### Q6：营业报表数据不对？

点击各板块的「🤖 刷新」按钮重新生成。如仍有问题，检查该日是否有未结账的订单。

### Q7：如何修改房间价格？

两种方式：
1. **单个房间**：房间管理 → 点击房间 → 编辑价格
2. **批量调价**：房价日历 → ⚡批量调价

### Q8：如何添加员工账号？

管理员登录 → 我的 → 用户管理 → 新增用户 → 选择角色。
